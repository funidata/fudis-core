# How to Contribute

Core Style Library is open for new development contributions in - and outside of Funidata company.

## Before contributing

All new design proposals are validated according to accessibility, visual consistancy and style re-usability.
Specially when proposing a new design token, modularity and re-usability are important factors.

In addition to code contributions, contributing may include style suggestions and documentation update requests.
Do not hesitate to contact Fudis team in advance for help and guidance.

## Contribution guidelines (external developer):

- Suggest your design proposal by [Contacting Fudis Team](/docs/documentation-introduction-welcome--docs#contact).
- **Implementation step-by-step**
  1. Get familiar with the Core development practices as documented in [Practices](/docs/documentation-development-practises-ways-of-working--docs).
  2. [Fork the repository](https://github.com/funidata/fudis-core/fork). Select yourself as the owner and press "Create fork".
  3. Set up your local development environment as documented in [Getting Started](/docs/documentation-development-getting-started--docs).
  4. Create a new branch, as guided in [Git and Github Convension](/docs/documentation-development-tooling-git-and-github-conventions--docs).
  5. Create your implementation including the following:
     - Design implementation
     - Visual regression tests
     - Storybook HTML Example
  6. Make a pull request on [Core Github page](https://github.com/funidata/fudis-core).
  7. Fudis team will review the PR, merge it or request for new changes. All open PRs are reviewed to satisfy the following acceptance criteria: accessibility, visual consistancy and modularity.

## Contribution guidelines (Funidata developer):

- Suggest your design proposal by [Contacting Fudis Team](/docs/documentation-introduction-welcome--docs#contact).
- Together with the Fudis team, will elaborate who is responsible for development.
- Implementation ticket is created on DS Backlog and it will be addressed to contributing developer.
